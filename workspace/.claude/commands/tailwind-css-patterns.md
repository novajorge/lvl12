# Tailwind CSS Patterns

## Overview
Expert in Tailwind CSS patterns and utilities for modern UI development.

## When to Use
Use this skill when working with Tailwind CSS, including:
- Building responsive layouts
- Creating custom components with Tailwind
- Optimizing Tailwind classes
- Using Tailwind utilities for animations, transitions

## Patterns Reference

### Layout
- Container: `container mx-auto px-4`
- Flexbox: `flex items-center justify-between`
- Grid: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3`

### Spacing
- Padding: `p-4`, `px-4 py-2`
- Margin: `m-4`, `mt-4 mb-2`

### Typography
- Text size: `text-sm`, `text-lg`, `text-xl`
- Font: `font-bold`, `font-medium`
- Colors: `text-gray-700`, `text-blue-500`

### Components
- Buttons: `px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600`
- Cards: `bg-white rounded-lg shadow-md p-6`
- Forms: `border border-gray-300 rounded px-3 py-2 focus:ring-2`

### Animations
- Transitions: `transition-all duration-300`
- Hover: `hover:scale-105 hover:shadow-lg`

## Best Practices
1. Use semantic class names
2. Keep utilities composable
3. Use responsive prefixes (md:, lg:)
4. Extract repeated patterns to components
